The SEDRIS Data Representation Model
APPENDIX B - Constraints
LSR Transformation Components

Definition

An <LSR Transformation> instance L shall have a <Local 4x4> component, an ordered set of <LSR Transformation Step> components, or both. If L has both a <Local 4x4> component and a set of <LSR Transformation Step> components, the ordered set of <LSR Transformation Step> components shall be mathematically equivalent to the <Local 4x4> component. If L is specified by a <Local 4x4> component only, it shall be possible to express L as an ordered set of <LSR Transformation Step> components.

Rationale

An <LSR Transformation> instance has two possible representations: as a <Local 4x4> component, or as an ordered set of <LSR Transformation Step> components. If the <LSR Transformation> instance uses both, the <Local 4x4> component and the <LSR Transformation Step> components are alternate representations of the same transformation, so they shall be mathematically equivalent. If the <LSR Transformation> instance uses neither representation, no transform is present, which means that the <LSR Transformation> instance should not exist.
