APPENDIX B - Constraints
Presentation Domain Constraints

Definition

The following constrains the use of <Presentation Domain> instances.

  1. In a given transmittal, every instance of <Colour>, <Image Mapping Function>, and <Rendering Properties> shall specify a <Presentation Domain> component, whether directly or through inheritance.

  2. In a given transmittal, every instance of <Image> that specifies a <Image Anchor> component shall specify a <Presentation Domain> component.

  3. Let X be a DRM object such that X has <Image Mapping Function> components F1 and F2. F1 shall specify a <Presentation Domain> component P1 and F2 shall specify a <Presentation Domain> component P2 such that the presentation domain sets of P1 and P2 do not intersect.

  4. Let X be a DRM object such that X has <Colour> components F1 and F2. F1 shall specify a <Presentation Domain> component P1 and F2 shall specify a <Presentation Domain> component P2 such that the presentation domain sets of P1 and P2 do not intersect.

  5. The presentation_domain field of a <Presentation Domain> instance shall not be empty.
