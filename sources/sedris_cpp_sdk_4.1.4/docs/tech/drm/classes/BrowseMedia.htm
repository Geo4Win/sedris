APPENDIX A - Classes
Browse Media

Class Name: Browse Media

Superclass - <SEDRIS Abstract Base>

Subclasses

This DRM class is concrete and has no subclasses.

Definition

An instance of this DRM class specifies a graphic or multimedia resource that provides a preview of the content of its containing DRM object.

Primary Page in DRM Diagram:

  • 20

Secondary Pages in DRM Diagram:

This class appears on only one page of the DRM class diagram.

Example

  1. A map of the area covered by a <Transmittal Root> instance or an <Environment Root> instance.

  2. A rotating image of a <Model> instance representing an M1A2 tank.

  3. A thumbnail version of an <Image> component of an <Image Library> instance.

  4. A data provider supplies a graphic visualization of a function defined by a <Data Table> instance by providing a <Browse Media> component for the <Data Table> instance.

  5. A data provider supplies a graphic depiction of the contents of a <Colour Table> instance by providing a <Browse Media> component for the <Identification> component of the <Colour Table> instance.

  6. A data provider with a movie clip of a fly-through of a transmittal represents the movie clip as a <Browse Media> component of the <Identification> component of the <Transmittal Root> instance.

FAQs

What is the purpose of this class?

This class provides ISO 19115-compliant metadata in the form of a graphical or multi-media representation that previews the content of its containing DRM object. Its purpose is to allow a potential user of the attributed object to easily assess the utility of the attributed object.

Constraints

  • <<Publishable Object>>
  • <<Mandatory Metadata>>

Component of (two-way)

  • zero or more <Identification> instances

Notes

Fields Notes

name

The name field specifies the title of the resource.

media_format

The media_format field specifies the media format of the resource (see Annex C, 2 Media_Format references).

media_urn

The media_urn field specifies a SEDRIS media URN (see SE_URN).

description

The description field, if not empty, provides a description of the resource.
