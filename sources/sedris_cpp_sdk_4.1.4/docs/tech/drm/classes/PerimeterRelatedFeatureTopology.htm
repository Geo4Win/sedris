Perimeter Related Feature Topology

Class Name: Perimeter Related Feature Topology

Superclass - <Feature Topology Hierarchy>

Subclasses

This DRM class is concrete and has no subclasses.

Definition

An instance of this DRM class is a <Feature Topology Hierarchy> instance that spatially organizes its components into a collection of (potentially) irregularly shaped regions, each defined by a <Perimeter Data> link object.

Primary Page in DRM Diagram:

  • 11

Secondary Pages in DRM Diagram:

This class appears on only one page of the DRM class diagram.

Example

  1. The geographic extent of a thematic coverage might be divided into a collection of tiles. Due to political boundaries or shorelines, some of these tiles might have irregular shapes. The <Feature Topology> instances contained within each tile are organized into components of a <Perimeter Related Feature Topology> instance.

FAQs

May the regions defined by the components of a <Perimeter Related Feature Topology> instance overlap?

No. The regions defined by the components of a <Perimeter Related Feature Topology> instance should fully partition the topological surface.

May the same <Feature Topology> instance be contained within more than one of the components of a <Perimeter Related Feature Topology> instance?

No. The organization of <Feature Topology> instances is strict, such that each <Feature Topology> instance shall be located within the region defined by a single component.
