Rendering Properties

Class Name: Rendering Properties

Superclass - <SEDRIS Abstract Base>

Subclasses

This DRM class is concrete and has no subclasses.

Definition

An instance of this DRM class specifies a selection of suggested shading algorithms, display sides, and presentation styles to use when rendering <Geometry Representation> instances.

Example

  1. A <Rendering Properties> instance specifies that flat shaded, solid, front-sides of polygons are to be rendered.

  2. A <Rendering Properties> instance specifies that interpolation shaded, solid, front-sides of polygons are to be rendered.

  3. A <Rendering Properties> instance specifies that Gouraud shaded, wire-frame, both-sides of polygons are to be rendered.

  4. A <Rendering Properties> instance specifies that Phong shaded, solid, back-sides of polygons are to be rendered. This case might be appropriate for a transmittal that describes an enclosed volume in which all of the polygons defining the enclosed volume face outwards, and the simulation occurs inside the enclosed volume. Thus, only the back sides of the polygons are to be rendered.

Notes

Composed of Notes


Presentation_Domain

If present, this component identifies the type of sensor for which the <Rendering Properties> instance is applicable.

Fields Notes

fill_method

The fill_method field specifies the colour blending method to be used.

shading_method

The shading_method field specifies the shading algorithm to be used.

colour_binding

The colour_binding field specifies the colour inheritance / overload rule in effect.

style

The style field specifies the set of SE_Display_Style members specifying what combination of wire and solid styles is to be used.

side

The side field specifies the set of SE_Display_Side members specifying which side(s) to display.

The front side of a <Polygon> instance is the side that has a counterclockwise orientation of the ordered <Vertex> components of the <Polygon> instance. Normally, either both sides or just the front sides of <Polygon> instances are rendered. Rendering just the front side of a <Polygon> instance culls the back face of the <Polygon> instance.
