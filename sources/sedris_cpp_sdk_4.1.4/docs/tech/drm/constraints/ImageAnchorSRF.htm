The SEDRIS Data Representation Model
APPENDIX B - Constraints
Image Anchor SRF

Definition

Consider an <Image Anchor> instance X.

  1. The <Location> components of X shall be valid or extended as allowed by the SRF specified by the srf_context_info field of X.

  2. The <Location> components of X shall specify distinct coordinate values that shall not be collinear.

  3. X shall be either a component of one or more <Image> instances or of one or more <Image Mapping Function> instances, but not both.

  4. If X is a component of an <Image Mapping Function> instance F, and if F appears within the scope of a DRM object that specifies an SRF S, the following shall apply.

    1. The srf_context_info field of X shall specify the same SRF as S.

    2. If X specifies a non-planar projection, the <Location> components of X shall specify the endpoints of orthogonal axes.

Rationale

Every <Location> instance is required to be valid within the context of the SRF in which it is defined. (In this sense, "valid" includes the SRM concept of "extended" coordinates.) To allow an <Image Mapping Function> instance specified with an <Image Anchor> component to be applied correctly to the textured objects, its SRF and theirs are required to be compatible.
