APPENDIX B - Constraints
Reference Surface Constraints

Definition

Consider a <Reference Surface> instance R.

  1. R shall appear in the scope of a DRM object that specifies a 3D SRF.

  2. R shall not appear in the component tree of any <Model> instance specified in an LSR SRF.

  3. Let S be the <Geometry Hierarchy> instance to which R is associated such that S specifies the reference surface of R.

    1. The SRF within which S is specified shall match that within which R is specified, unless S is a <Property Grid Hook Point> instance.

    2. If S is a <Property Grid Hook Point> instance, S shall have at least one <Property Grid> component G such that the following apply.

      1. The SRF within which S is specified shall match that specified by G.

      2. G shall have two spatial <Axis> components corresponding to the primary and secondary coordinates of the SRF specified by G.

      3. G shall have a <Table Property Description> component for height, elevation, or bathymetry.

      4. The qualified classification specified by G shall match that specified by R.

    3. If S is a <LOD Related Geometry> instance, the lod_data_type field of S shall not have the value SE_LODDATTYP_VOLUME.
