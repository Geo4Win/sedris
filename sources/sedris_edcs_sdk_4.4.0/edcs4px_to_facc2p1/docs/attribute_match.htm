Concept Mapping from EDCS 4.3 to FACC 2.1 - Attribute Mapping Table

Caveats on the interpretation of the EDCS 4.3 Concept Mapping Type ONE_TO_ONE:

  • The EDCS Edition 4.3 Attribute Dictionary supports the ability for attributes which are COUNT, INTEGER, and REAL values to be both singletons, and intervals. The FACC Edition 2.1 only supports the ability to represent single values (either "integer" or "floating point") -- a change in datatype is effectively occuring. Therefore, in some cases it may not be possible to map ONE_TO_ONE, depending on the actual attribute instance value; in those cases the EDCS 4.3 Concept Mapping Type is identified as INDETERMINATE.
    This indicates that while the FACC attribute identified in the FACC 2.1 Concept Mapping Information applies if the EDCS attribute instance is single-valued, an alternative FACC attribute may apply if the EDCS attribute instance is interval-valued or metadata -- in which case the necessary mapping information can be found in the appropriate Enumerant Mapping Tables for that EDCS attribute.

  • The EDCS Edition 4.3 allows for attribute concept instances to not have a value, but instead have an Attribute Value Metadata value. The FACC Edition 2.1 provides for metadata to be explicitly represented where a "coded list" is provided, although the actual binding of code values to metadata semantics may vary from attribute-to-attribute. Table 5.2 Special Reserved Values indentifies the standard mechanisms FACC uses to denote metadata for all attribute value representations. It is the responsibility of the application to recognize the presence of EDCS attribute value metadata and to use the appropriate reserved value for the indicated FACC attribute for attributes whose values are represented as text, integer (short and long), or floating point (single or double precision).
